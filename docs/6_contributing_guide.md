# 6. Guide de Contribution

Contribuer à Musify est une excellente manière de participer à un projet open-source dynamique. Toute aide est la bienvenue, que ce soit par le code, les traductions, les rapports de bugs ou les suggestions.

## Principes de Base

Avant de contribuer, il est important de prendre connaissance des documents qui régissent la communauté du projet.

*   **[Code de Conduite (Code of Conduct)](../CODE_OF_CONDUCT.md)**: Ce document établit les règles pour un environnement collaboratif respectueux et positif. Tous les contributeurs sont tenus de le suivre.

*   **[Guide de Contribution (Contributing Guide)](../CONTRIBUTING.md)**: Ce fichier contient des instructions techniques détaillées sur :
    *   Comment configurer votre environnement de développement.
    *   Le style de code à adopter.
    *   La manière de soumettre une *Pull Request*.
    *   Les informations à inclure dans les rapports de bugs et les demandes de fonctionnalités.

*   **[Licence](../LICENSE)**: Le projet est sous licence **GPL v3.0**. En contribuant, vous acceptez que vos contributions soient distribuées sous cette même licence.

## Comment Aider ?

*   **Rapporter des bugs**: Si vous trouvez un problème, ouvrez une "Issue" sur GitHub en utilisant le modèle de rapport de bug.
*   **Suggérer des fonctionnalités**: Utilisez le modèle de demande de fonctionnalité pour proposer de nouvelles idées.
*   **Traduire l'application**: Aidez à traduire Musify dans votre langue.
*   **Écrire du code**: Corrigez des bugs ou implémentez de nouvelles fonctionnalités. Assurez-vous de suivre les directives du `CONTRIBUTING.md`.
